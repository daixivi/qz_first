假设你和朋友们一起写一篇文章（相当于一起开发一个软件/网页）：

你们有一个主版本（大家共同的最终稿），叫 main（以前叫 master）。

现在你想尝试加一个新段落，但又怕直接加到主版本会改乱，于是你另开一个副本去写。

这个副本就是分支（branch）。

你在副本里随便改、随便试，不会影响主版本的内容。

等你改好了、确认没问题，再把副本的改动合并回主版本。

 

2️⃣ 在 GitHub 项目里的实际意义

main 分支：项目的“正式版”，一般用来发布（比如我们网页上线就用 main 分支里的文件）。

其他分支：比如  dev （开发）、 feature/新功能 （做某个功能）、 fix/bug （修 bug）。

每个分支其实就是一份独立的代码快照，互不影响，你可以在不同分支做不同实验。

举例（我们目前的简单项目）：

我们只有一个  main  分支，里面放着  index.html 。

如果我们想试试新页面，可以新建一个分支  test-page ，在里面写新文件，而不影响  main  里的网页。

等测试完再合并到  main ，正式上线。

 

3️⃣ 为什么要有分支？

保护主版本：避免直接在正式版上改出 Bug。

多人协作：不同的人在不同的分支开发，最后合并到一起。

功能实验：可以大胆试新功能，失败就删掉分支，不影响主线。
