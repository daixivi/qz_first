打开 https://github.com ，注册账号（用邮箱）。

登录后，点右上角 “+” → New repository。

填仓库名（如  my-first-web ），选 Public，
勾选 Add a README file → Create repository。



在仓库页面点 Add file → Upload files，把  index.html  拖进去，Commit changes。

方法 B（用 Git 命令行）：后面可以学，现阶段网页上传更简单。




在仓库里点 Settings → Pages（左侧菜单）。

Source 选 main branch（或 master），点 Save。

Source两种选项
Deploy from a branch	从分支部署	直接从仓库的某个分支
（如  main  或  gh-pages ）读取文件并生成网页。
这是最常见、最简单的发布方式。	初学者、静态网页（HTML/CSS/JS）

GitHub Actions	用自动化工作流部署	
用事先写好的自动化脚本（YAML 文件）来控制怎么构建和部署网页。
适合需要编译、打包的项目（如 React/Vue 项目）。	有 CI/CD 经验的开发者




企业正规开发文件管理布置
my-first-web-pro/
├── docs/                   # GitHub Pages 发布目录（网站文件在这里）
│   ├── index.html          # 首页
│   ├── about.html          # 其他页面（如果有）
│   ├── css/
│   │   └── style.css       # 样式文件
│   ├── js/
│   │   └── main.js         # 脚本文件
│   └── assets/
│       └── logo.png        # 图片等资源
├── src/                    # 源代码目录（开发时编辑的地方）
│   ├── index.html
│   ├── css/
│   │   └── style.css
│   ├── js/
│   │   └── main.js
│   └── assets/
│       └── logo.png
├── .gitignore              # Git 忽略文件列表
├── README.md               # 项目说明文档
└── LICENSE                 # 开源许可证（可选）


首页只能是index.html这个名字，并且按照现在的github，只能放在根目录或者docs目录下面
创建github pages的时候folder也只能选根目录或docs


整个文件夹丢进github就行了
settings page只是配置你的这个仓库的网址

这个强制css，不然scss有时候会报错，加在head里
<link rel="stylesheet" href="assets/css/style.css">
